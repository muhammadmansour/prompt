# WathbahGRC Admin — Typical Evidence Management
**Snapshot taken:** 2026-04-14
**Commit / branch:** a03a947 / db-scheme-changes
**Scope of this file:** Admin surface for managing typical evidence items.

## Status: PRESENT (embedded in Audit Studio flow)

Typical evidence items are generated alongside questions during requirement analysis. They follow the same lifecycle as questions.

### UI Route
- Legacy: `index.html` → analyze → modal shows "Typical Evidence" section with edit/add/delete per item
- Modern SPA: Part of the chat-based audit flow output (free-form in chat response)

### Backend
- Generated by `callGeminiAPIForSingle()` as `typical_evidence` array (`server.js:1148`)
- Written to Muraji API as `typical_requirements` (a flattened bullet-point string) via the frontend (`app.js:1088–1096`)

### Limitations
- Same as questions: no standalone CRUD, no bulk operations, no evidence catalog
- Evidence is stored as a **plain text string** (bullet list) on the Muraji API, losing the structured `{title, description}` format
- No file attachment capability for evidence items
- No evidence status tracking (pending, accepted, rejected)
