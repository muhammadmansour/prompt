# WathbahGRC Admin — Question Management
**Snapshot taken:** 2026-04-14
**Commit / branch:** a03a947 / db-scheme-changes
**Scope of this file:** Admin surface for managing questions attached to requirements.

## Status: PRESENT (embedded in Audit Studio flow)

Questions are generated by AI during the Audit Studio requirement analysis flow and can be manually edited before saving.

### UI Route
- Legacy: `index.html` → select requirements → analyze → modal shows questions → confirm writes to Muraji API
- Modern SPA: The chat-based audit flow (`/audit-studio`) does not have a dedicated question management screen. Questions appear as part of the initial AI chat response in free-form markdown.

### Backend Endpoint
- `POST /api/analyze` (`server.js:1901`) — triggers Gemini analysis that returns questions
- Writeback: Frontend calls `PATCH https://muraji-api.wathbahs.com/api/libraries/:id/controls` directly (no server proxy)

### Permission Check
- Auth guard only (any logged-in user)

### Limitations
- No standalone CRUD for questions — they can only be created/edited during an analysis session
- No listing of all questions across requirements
- No bulk question operations
- No question templates or question banks
- Questions-to-controls conversion exists: `POST /api/controls/from-question` (`server.js:1990`)
