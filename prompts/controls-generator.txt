## Instructions

Generate 1–3 candidate applied controls for the above requirement, tailored to the organization's industry vertical, size, compliance maturity level, regulatory mandates, governance structure, data classification, and infrastructure type.

**Tailoring rules:**
- Low maturity orgs (1–2): simpler, foundational controls; fewer controls per requirement
- High maturity orgs (4–5): advanced, layered controls; consider automation and continuous monitoring
- Sector-specific: inject industry-specific practices (e.g. PCI DSS for banking, HIPAA for healthcare)
- Size-aware: enterprise orgs need formal governance; small orgs need pragmatic lightweight controls
- Data classification: Restricted/Confidential data demands stricter controls
- Cloud-native vs on-premise vs hybrid: adapt technical implementation guidance accordingly

Each control MUST include ALL of these fields:
1. **name**: A clear, concise control name (English, 5-15 words)
2. **name_ar**: Arabic translation of the control name
3. **description**: Detailed description of what the control entails, how to implement it, and what evidence demonstrates it (English, 30-80 words)
4. **description_ar**: Arabic translation of the description
5. **category**: One of: "policy", "process", "technical", "physical" — the nature of the control
6. **csf_function**: One of: "govern", "identify", "protect", "detect", "respond", "recover" — which NIST CSF 2.0 function this control maps to
7. **status**: Always "to_do" (controls start as not-yet-implemented)
8. **priority**: Integer 1–4 where 1 = Critical/Urgent, 2 = High, 3 = Medium, 4 = Low — based on risk and compliance gap
9. **effort**: One of: "S", "M", "L", "XL" — estimated implementation effort for this org size/maturity (S=Small/days, M=Medium/weeks, L=Large/months, XL=Extra-large/quarters)
10. **relevance_score**: Integer 0–100 indicating how relevant this control is to the specific organization (100 = perfectly tailored, 50 = generic)
11. **evidence_examples**: Array of 2-3 specific evidence items that prove this control is in place
12. **implementation_guidance**: Brief practical implementation guidance specific to the org's infrastructure and governance (2-3 sentences)
